Error

(e) All errors are reported using this format (with the HTTP-status code matching the number)

error(1)

        (e1) :{"code": 404, "message": "The Person entity with ID: '5' was not found"}

error(2)

        (e2) :{ status : 404, "message": "Zipcode: \u00270001\u0027 was not found" }

error(3)

        (e3) :{ status : 404,   "message": "Phone number: 123 already exists in the database" }

error(4)

        (e4) :{ status : 404,   "No CityInfo entities exist. Call '.../api/cityinfo/populate' to populate the database with valid CityInfo data" }

error(5)

        (e5) :{ status : 404,   "No Hobby entities exist. Call '.../api/hobby/populate' to populate the database with valid Hobby data" }
